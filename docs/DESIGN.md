# 重生投研部 · 叙事设计文档

这份文档说明这个游戏的叙事系统为什么这样设计、已经落地了哪些机制、以及内容生产成本怎么压到最低。它写给后续维护者、写手和自动化助手看。代码层面的具体字段以 `src/types.ts` 为准，本文只讲设计意图和对应关系。

## 1. 这个游戏到底是什么

这是一个金融职场叙事游戏，也可以当成 CFA 知识的内化平台。主角 顾行之 是一个穿越者，脑子里带着三年后的世界走向，但本人没有任何金融专业技术。他被空降回入职第一年的投研部，身边有三个同事：

- 林若宁，基本面研究员，信产业链上下游交叉验证。
- 陈星禾，量化与资金信号研究员，信大单流向、因子拥挤度和波动率曲面。
- 周明昭，宏观与风险研究员，信仓位、流动性和尾部风险。

核心命题只有一个：知道未来，不等于理解现在。玩家手里有结论，但要把结论讲成一个专业同事听得懂、委员会敢采纳的研究，得靠这一年里一点点学回来的手艺。

## 2. 为什么要改

初版的问题不在题材，在于三处让叙事和玩法脱节：

- 学习是被动的。主角被同事喂知识，玩家只是看客，没有上手的机会。
- 穿越是装饰性的。三年记忆只用来剧透，不用来制造两难和代价。
- 循环太薄。每个月只是换汤不换药地选一次，选择之间没有积累，没有回响。

改法的方向不是加更多剧情，而是把已有的设定变成机制：记忆变成一种负担，知识变成可以随身携带的卡牌，办公室变成一个会随时间变化的地方，选择会在几个月后回来找你。

## 3. 八条非共识设计原则

下面这些不是行业常识，是我们在这个项目里特意坚持的。

### 3.1 记忆是负债，不是外挂

大多数穿越文把先知当成作弊码。我们反过来：顾行之 随口丢出一个正确结论，但没有证据链支撑，同事会觉得他飘、委员会会警惕。引擎里有一个空降结论检测，玩家分数高但证据分和清晰分双低时触发，记下一个 `parachuted_${framework}` 标记。

### 3.2 没有唯一正确答案

同一个未来记忆，三个同事会各自推出一套自洽的判断。林若宁看生意，陈星禾看资金，周明昭看风险。游戏不在结局告诉你谁对，玩家要在分歧里学会权衡。每月的 `competingHypotheses` 就是给这三套视角留的接口。

### 3.3 教学在角色口中说，不在弹窗里

知识卡是同事用自己的语气说的一句话，不是教科书弹窗。`KnowledgeCard.mentorLine` 必须是这个角色会讲的话，不是作者旁白。这样学习内容和人物塑造是同一件事。

### 3.4 办公室是第四位角色

办公桌上的便签、白板上的框架草图、堆起来的咖啡杯，都是叙事载体，不是装饰。`OfficeState` 按月累积这些道具，让时间有了可见的痕迹，也省掉了大段旁白。

### 3.5 失败要安全

所有评级和分支都发生在模拟环境里，没有真钱、没有不可逆的社交死亡。玩家可以放心地选错，因为错本身也是内容。安全失败才能换来真实验证。

### 3.6 评级驱动分支

每月结算出 S 到 D 的评级，对应三个维度：尊重、观察、负债。高评级点亮 `respect_${framework}`，低评级点亮 `watch_${framework}`，空降结论点亮 `parachuted_${framework}`。后续月份的分支读取这些标记，给回头客不同的对话和机会。

### 3.7 延迟后果

早期帮 陈星禾 改模型、补数据，会埋下一个 `helped_xinghe` 种子。要到第 8 个月之后，它才兑现成一次旁听闭门会的席位。玩家当月感觉不到，但世界记住了。

### 3.8 不做割裂的双模式

不做一个剧情模式加一个答题模式的拼凑。叙事、选择、知识获取是同一层体验，玩家始终在角色里，不会被踢出戏去考试。

## 4. 已落地的机制（与代码一一对应）

### 4.1 知识卡 KnowledgeCard

定义在 `src/types.ts`。每次有意义的决策后，引擎调用 `pickKnowledgeCard`（`src/game/content.ts`）从 `MENTOR_TEACHINGS` 里挑一张卡。优先用决策自己带的 `teaches`，否则按该决策的框架归属加类别推导。卡牌进 `GameState.knowledgeCards`，去重按 `id`。

`MENTOR_TEACHINGS` 目前覆盖三位导师乘八种类别，教学台词都按角色语气写。卡牌分三层：

- tier 1，表面层，主线必看。
- tier 2，投入层，认真玩家才会集齐。
- tier 3，深挖层，留给考据型玩家。

### 4.2 业务事实结算 businessOutcome

这是关键约束：股价不作为真相仲裁。短期价格充满情绪噪声，不能代表公司本质。所以每个月的结算文案写的是业务现实，例如产能落地、订单节奏、监管动作，而不是涨跌幅。

`MarketTheme.businessOutcome` 是作者手写的确定性文案，可控、诚实、能容纳多种结果。复盘面板里的「业务事实结算」块直接展示它。

### 4.3 三方竞争假说 competingHypotheses

每月场景里，引擎在满足条件下注入一个 `competingNode`（`buildCompetingNode` 生成），把林若宁、陈星禾、周明昭对同一件事的不同判断摆到玩家面前。它从第 2 个月起出现，前提是当月主题写了 `competingHypotheses`。这个节点让「没有唯一正确答案」从口号变成玩家真的会看见的东西。

### 4.4 记忆负债检测 isParachuted

在 `src/game/engine.ts` 的 `makeDecision` 里：

```
const isParachuted = score.evidenceScore + score.clarityScore < 24 && score.total >= 70;
```

意思是总分高但证据和清晰双低，说明靠先知蒙对了。命中后亮标记，后续相关导师的对话会带上警惕口吻。

### 4.5 评级与种子驱动分支

`src/game/content.ts` 里 `GRADE_BRANCHES` 给三位导师各生成九条一次性分支，分别读 `respect_`、`watch_`、`parachuted_` 标记，给回头客专属反馈。`DEBT_BRANCH` 读 `helped_xinghe` 且月份大于等于 8，兑现闭门会席位。所有这些分支都用 `once: true` 和 flag 门控，所以测试态和全新存档下不会乱触发，现有测试保持绿色。

### 4.6 办公室作为第四角色 OfficeState

`OfficeState` 有四个计数：便签数、白板笔迹数、咖啡杯数、已过月数。引擎在 `makeDecision` 里按决策类别累加，例如深度研究和帮同事会加便签，深度研究、拆数据、投委会答辩会加白板笔迹，熬夜会加咖啡杯。`src/App.tsx` 里的 `OfficeWidget` 把这些计数渲染成有温度的文案。

### 4.7 复盘与笔记本 UI

`src/components/StoryRecapPanel.tsx` 重写后展示四块：同事评分反馈、事后复盘、业务事实结算、当月知识卡。右侧栏还加了 `ResearchNotes`（玩家的研究笔记，列出已收集的知识卡）和 `OfficeWidget`（办公室状态）。`src/components/EndingPanel.tsx` 在结局页展示学了几条手艺。

## 5. 内容生产规范

每月内容是一个 `MarketTheme`，必须尽量填三个新字段：

- `knownEvent`：顾行之 带回去的、将来会发生的业务事件。序章用它埋钩子。
- `businessOutcome`：该业务事件落地后的真实情况，作者手写，不用股价。
- `competingHypotheses`：三套互相能自洽的判断，分别给林若宁、陈星禾、周明昭。

`src/game/content2025.ts` 已经把 2025 年十二个月全部补上这三块，验证通过后通过 `THEMES_2025` 暴露。后续年份照同样结构补即可。

八种决策类别固定为：深度研究、专家访谈、路演、风险提示、自我照顾、帮同事、投委会答辩、拆数据。写手在补类别教学或业务角度时，必须落到这八个桶里。

## 6. 玩家一局体验

一局十二个月。每月流程：

1. 看当月主题与钩子，序章已被 `knownEvent` 埋过线。
2. 选一个聚焦方向，再在一个决策点做选择。
3. 结算评级，看到同事复盘、业务事实、当月知识卡。
4. 办公室道具默默累加，笔记里多一张卡。
5. 某些月份会撞见三方竞争假说，被迫在分歧里表态。
6. 早期的选择在第 8 个月后陆续回来，变成席位、信任或警惕。
7. 年末看结局，手艺数量、关系、办公室状态共同定义这一年的你。

## 7. 关键设计约束

- 股价永不作为真相仲裁，只用业务事实结算。
- 关键路径不依赖任何选修知识卡，没集卡也能通关看懂。
- 每个分支选择要在两个场景内产生可感知后果，否则这个选择等于没有。
- 所有新增分支必须 flag 门控且 `once: true`，不能污染测试态和全新存档。
- 教学台词必须像这个角色会说的话，不能是作者借角色之口讲课。

## 8. 后续可扩展

- 环境变量叙事：把咖啡杯、便签数接进场景背景渲染，让办公室真的随玩家变化。
- 知识卡图鉴：结局后开放完整 glossary，按 tier 标收集度。
-  localization：所有角色台词走字符串外置，方便后续多语言。
- 数据驱动复盘：把已发布的市场复盘 JSON 接到 `businessOutcome` 的生成链路，减少手写成本。

## 9. 文件地图

- `src/types.ts`：所有叙事相关类型，含 `KnowledgeCard`、`OfficeState`、`CompetingHypotheses`、`DecisionCategory`。
- `src/game/engine.ts`：纯逻辑引擎，`makeDecision` 里算评级、知识卡、空降检测、业务结算、办公室累加。
- `src/game/content.ts`：`MENTOR_TEACHINGS`、`pickKnowledgeCard`、`GRADE_BRANCHES`、`DEBT_BRANCH`、`buildCompetingNode`。
- `src/game/content2025.ts`：2025 年十二个月主题，含 `knownEvent`、`businessOutcome`、`competingHypotheses`。
- `src/components/StoryRecapPanel.tsx`：复盘面板，展示四块内容。
- `src/App.tsx`：`ResearchNotes` 与 `OfficeWidget` 组件，挂在右侧栏。
- `src/components/EndingPanel.tsx`：结局页展示手艺数量。
