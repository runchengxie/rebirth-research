# 研究平台模式与扩展契约

本文说明主菜单、年度剧情体验、独立投委会、每日挑战、社区内容工坊、叙事状态机和加密云同步。它们共用研究承诺、研究方案与业务事实模型，避免每个模式维护一套互不兼容的评分语言。

## 模式入口

无查询参数访问显示主菜单。主菜单把继续游戏、年度剧情新游戏、挑战中心和创作入口分组，不在每个玩法底部常驻四项模式栏。

主菜单顶部使用 `assets/key-art.webp` 作为封面。封面只提供氛围，继续游戏和模式入口都位于独立的实色区域。

| 目标 | URL | 用途 |
|---|---|---|
| 主菜单 | 默认 | 继续本地进度、开始新游戏或选择独立玩法 |
| 年度剧情 | `?mode=story` | 恢复当前存档的十二话剧情与人物路线 |
| 投委会 | `?mode=committee` | 选择案例并完成五轮独立答辩 |
| 每日挑战 | `?mode=daily` | 按本地日期生成所有玩家相同的案例与约束 |
| 内容工坊 | `?mode=studio` | 创建、验证、导入、导出和本地安装社区案例包 |

三个独立平台模式通过动态导入加载。主菜单和年度剧情不下载内容工坊与完整答辩界面。进入任一玩法后，返回主菜单入口由轻量应用外壳提供。

年份、Pixi'VN 原型和舞台参数是已有深链。含有 `year`、`pixivn`、`pixi` 或 `staticStage` 时继续直接进入年度剧情，旧书签不会被主菜单截断。

## 年度剧情体验

年度剧情内部使用独立的 `ExperienceMode`，不把玩法难度塞进 `PlatformMode`：

| 体验 | 新游戏 URL | 主界面规则 |
|---|---|---|
| 剧情模式 | `?mode=story&play=romance&new=1` | 隐藏职业指标、调查、研究线索、日程和研究承诺，突出对白与人物关系 |
| 职业模式 | `?mode=story&play=career&new=1` | 保留完整调查、研究承诺、评分、职业结局和因果回溯 |

`play` 提供新存档的体验默认值。`new=1` 要求会话层跳过已有进度并创建新状态，创建完成后从 URL 移除。继续游戏读取原子会话存档或 `rebirthMeta:v4` 中的 `experienceMode`，旧 v3 存档没有该字段时迁移为职业模式。

剧情模式的简化发生在两个边界。React 策略控制可见界面，`sessionMachine.ts` 在结算前自动补充稳健研究承诺并缓和隐藏的疲劳与生活平衡惩罚。职业模式直接使用玩家填写的研究承诺。两种体验继续使用同一剧情数据、关系旗标和结算引擎。

## 默认剧情状态机

年度剧情由 `src/app/useGameSessionMachine.ts` 和 `src/game/sessionMachine.ts` 驱动。

`GameSessionAction` 包含：

- 替换年份快照。
- 重新开始。
- 推进、回看和跳过已读。
- 整理研究室。
- 选择日程。
- 执行调查。
- 提交研究判断。
- 创建或恢复时间线分支。
- 执行反事实推演。

所有行动通过一个状态归约器同时返回 `GameState` 与 `RebirthMetaState`，再统一写入旧版兼容存档和原子会话存档。界面组件不再分别拼接两套状态更新。

`RebirthMetaState.experienceMode` 在新游戏创建后保持稳定。重新开始、进入下一周目和恢复时间线都会沿用当前体验，避免一份路线在中途更换隐藏规则。

`src/game/narrativeMachine.ts` 把当前节点解释为以下显式阶段：

- `dialogue`
- `debate`
- `decision`
- `result`
- `ending`

阶段类型只负责描述主流程，不替代现有内容节点和时间线锚点。以后增加新的渲染器时，应先扩展叙事阶段或节点契约，避免继续依赖散落的字符串后缀判断。

## 独立投委会

投委会模式从 2025 年正式内容和本地社区案例库读取案例。

玩家先完成：

1. 选择研究方案。
2. 声明置信度。
3. 指定失效条件。
4. 完成证据、反例和退出边界自检。

随后接受五轮追问：

| 轮次 | 角色 | 主要检查 |
|---|---|---|
| 资源配置 | 基金经理沈砚川 | 判断为何值得占用风险预算 |
| 证据链 | 林若宁 | 最可能推翻核心假设的证据 |
| 模型归因 | 陈星禾 | 样本、动量和失败区间 |
| 下行情景 | 周明昭 | 错误后的代价与退出条件 |
| 信息边界 | 合规负责人许闻舟 | 事实、推断和未来记忆来源 |

答辩按以下维度结算：

- 证据。
- 清晰度。
- 风险意识。
- 沟通。
- 研究诚信。
- 置信度校准。

完整表达不会自动获得高分。隐藏失败样本、用个人权威替代来源、在证据不完整时给出高确信，都可能换来短期沟通分并损害诚信与风险评分。

答辩历史最多保存最近 30 条，只存在当前浏览器。

## 程序化角色与场景

新增组织角色当前服务于投委会模式：

- 沈砚川：基金经理，负责组合风险预算和行动价值。
- 许闻舟：合规负责人，负责公开来源、推断边界和表达纪律。

他们暂时不进入恋爱、研究图鉴或年度关系数值，避免把组织功能角色强行改造成另一条好感度进度条。

答辩场景使用 CSS 程序化构图：

- 投委会会议室。
- 数据墙。
- 风险审查室。
- 合规桌面。

这些场景不依赖新增位图，可以随主题和设备尺寸自适应。后续替换为正式美术资源时，应保持相同的场景编号，避免内容包和答辩逻辑跟随资源路径变化。

## 每日挑战

每日挑战使用本地日期字符串作为确定性种子。

同一天、同一版本的内容会得到：

- 相同案例。
- 相同组织约束。
- 相同五轮问题。

当前没有中心排行榜，也不上传结果。这样可以在纯静态部署下提供全员同题体验，同时避免为了一个分数榜引入账号、反作弊和隐私基础设施。

浏览器保存最近 90 天结果，并计算连续完成天数。首次完成当天结果后，可以进入练习模式，练习不会覆盖当天首次记录。

内容版本变化可能改变同一日期映射到的案例。每日结果因此记录完成时的案例编号、标题、方案与分数，不依赖之后重新计算。

## 社区内容包

内容包格式：

```json
{
  "format": "rebirth-research-community-pack",
  "version": 1,
  "id": "pack-id",
  "title": "案例包标题",
  "author": "作者",
  "description": "说明",
  "updatedAt": "ISO 时间",
  "cases": []
}
```

每个案例必须包含：

- 标题。
- 公开信息。
- 未来记忆。
- 基本面、量化和风险三种假设。
- 业务事实结算。
- 至少两个研究方案。

每个研究方案必须包含：

- 稳定且唯一的 `id`。
- 名称和说明。
- 研究方法。
- 执行质量。
- 证据、清晰、风险和反思分值。

内容工坊负责：

- 创建和编辑内容包。
- 实时结构校验。
- 案例与三种假设预览。
- JSON 文件和粘贴文本导入。
- JSON 导出。
- 保存到浏览器本地案例库。
- 删除已安装内容包。

本地保存后的案例会直接出现在投委会案例库。社区内容不进入年度剧情存档，不允许任意脚本、HTML、远程资源或函数，只保存受限的结构化文本和数字字段。

### 版本规则

- `format` 用于识别产品。
- `version` 用于迁移结构版本。
- 修改字段语义时必须升级版本。
- 增加可选字段时可以保留当前版本，但需要保证旧内容仍可读取。
- 导入失败必须展示具体字段错误，不能静默补全为看似有效的案例。

## 加密云同步

项目仍然没有自己的账号服务器。可选云同步使用玩家自己的私密 GitHub Gist。

### 上传流程

1. 从当前年份读取完整存档包。
2. 浏览器生成随机盐值和初始向量。
3. 使用 PBKDF2-SHA-256 从同步口令派生 256 位密钥。
4. 使用 AES-GCM 加密存档文本。
5. 将密文写入私密 Gist 的 `rebirth-research-save.json`。

### 拉取流程

1. 使用 GitHub 访问令牌读取指定 Gist。
2. 校验加密文件格式。
3. 在浏览器内使用同步口令解密。
4. 校验解密后的游戏存档。
5. 写入本地单周目、跨周目和原子会话存档。

### 安全边界

- GitHub 访问令牌只存在当前输入框状态，不写入 `localStorage`。
- Gist 编号可以按年份保存在浏览器，便于下次更新。
- 同步口令不保存，也无法找回。
- Gist 中只保存加密密文、盐值、初始向量、算法参数和更新时间。
- 项目无法替用户撤销泄露的 GitHub 访问令牌。
- 浏览器扩展、恶意脚本或被入侵的设备仍可能读取输入框内容，客户端加密不等于设备安全。
- 使用私密 Gist 仍受 GitHub 服务条款、配额和可用性约束。

多用户账号、服务端冲突合并、密钥恢复和跨设备自动推送需要单独建设后端。当前方案只提供由玩家自行保管访问权限的加密云端存储。

## 主要代码位置

- `src/components/StartMenu.tsx`：继续、新游戏、挑战中心和创作入口。
- `src/components/BackToMenu.tsx`：玩法页面返回主菜单入口。
- `src/game/platformModes.ts`：主菜单目标、兼容深链与 URL 生成。
- `src/game/experienceMode.ts`：剧情模式和职业模式策略。
- `src/game/sessionMachine.ts`：年度剧情原子状态转换。
- `src/app/useGameSessionMachine.ts`：React 状态归并、存档与音频适配。
- `src/game/narrativeMachine.ts`：显式叙事阶段。
- `src/game/committeeMode.ts`：答辩问题、回答和评分。
- `src/modes/CommitteeMode.tsx`：独立投委会界面。
- `src/game/dailyChallenge.ts`：确定性每日案例、历史和连续天数。
- `src/modes/DailyChallengeMode.tsx`：每日挑战界面。
- `src/game/communityContent.ts`：内容包结构、校验和本地案例库。
- `src/modes/ContentStudioMode.tsx`：可视化内容编辑器。
- `src/game/cloudSync.ts`：客户端加密和 GitHub Gist API。
- `src/components/CloudSyncPanel.tsx`：云同步设置与操作。
- `src/start-menu.css`：主菜单响应式样式。
- `src/platform-shell.css`：玩法返回入口与云同步所需的轻量样式。
- `src/platform.css`：随平台模式按需加载的完整界面与程序化场景。
