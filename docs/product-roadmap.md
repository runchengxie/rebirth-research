# 产品改进路线图

本文是 `docs/` 中唯一记录未来候选工作的文档。其他玩法、界面和架构文档继续只描述已经实现的行为。路线图最后核对日期为 2026 年 7 月 16 日。

这轮改进的目标有三个：降低职业模式在手机上的操作负担，让组织压力真正进入剧情因果，让场景、办公室和角色方法论共同承担叙事。现有剧情模式、职业模式、四名常驻角色、评分引擎和存档格式继续作为稳定基础。

## 使用方式

- `[ ]` 表示尚未开始。
- `[x]` 表示已经完成并通过对应验收。
- 开始任务时，在条目末尾补充负责人或拉取请求链接。
- 任务改变当前行为后，同步更新对应的现状文档，不能只勾选本路线图。
- 每个阶段可以拆成多个小拉取请求。前置验收没有通过时，不进入依赖它的后续阶段。
- 规模使用 `S`、`M`、`L` 表示相对工作量，只用于排序，不作为工期承诺。

## 当前实现基线

以下能力已经存在，后续工作应在这些基础上扩展：

- 年度剧情已经分为剧情模式和职业模式，内部编号为 `romance` 与 `career`。
- 剧情模式已经隐藏调查、研究线索、日程、研究承诺和职业指标，并由系统补充稳健研究承诺。
- 职业模式已经计算完整月度评分，并把评分写入历史记录。
- 2025 十二个月已经有基金经理、销售、负责人、合规、客户等组织压力，但目前只显示为静态压力卡。
- 独立投委会已经有基金经理沈砚川、合规负责人许闻舟和五轮答辩，不需要在年度主线重复建设。
- `OfficeState` 已经记录便签、白板、咖啡杯和月份，研究室抽屉已经提供动态概览、物件整理和二周目残留线索。
- 常规舞台已经注册研究室、会议室和夜间咖啡馆，正式主线目前只实际使用前两类。
- 四名常驻角色已经有 `methodology` 描述，舞台仍主要依赖颜色、立绘和姿势区分方法。
- 主舞台已经移除逐件计数的办公室装饰，后续环境反馈不能恢复这种遮挡方式。

## 长期边界

后续实现都需要遵守以下边界：

- 保留现有 `CharacterId` 四人集合。组织角色不得进入关系值、攻略路线、知识卡导师表或恋爱结局。
- 保留内部 `romance | career`、现有 URL 参数和存档迁移路径。暂不增加第三种年度体验枚举。
- 继续复用现有评分引擎。展示评分拆解时不修改权重、阈值和历史结果。
- 场景编号只描述场所与呈现方式，不承担评分或分支逻辑。
- 组织后果优先复用 `committeeAdoption`、`clientFeedback`、`teamTrust`、`researchCredibility` 和稳定旗标。
- 办公室主舞台反馈由现有状态派生，不新增持久化计数。
- 选择前解释只能说明术语含义，不能暗示正确方案、角色好感或评分结果。
- 所有核心流程继续支持纯静态部署、离线运行、静态舞台降级和减少动态效果。

## 推荐推进顺序

| 阶段 | 目标 | 规模 | 主要依赖 |
|---|---|---:|---|
| R0 | 修复测试盲点并固定架构边界 | S | 无 |
| R1 | 手机端步骤式职业决策 | M | R0 |
| R2 | 术语解释、评分拆解和方法档案 | S 至 M | R0，可与 R1 并行 |
| R3 | 场景注册表、降级一致性和低成本复用 | M | R0 |
| R4 | 组织角色媒介与关键月回调试点 | M 至 L | R2、R3 |
| R5 | 角色方法论视觉与办公室环境状态 | M 至 L | R3 |
| R6 | 持续职业弧、新场景位图和内容格式升级 | L | R4、R5 的试玩结论 |

R1 与 R2 可以并行。R4 先做四至六个月试点，验证有效后再进入 R6。美术资源生产应晚于 R3 的场景资源契约，避免新增资源继续散落在多个渲染路径中。

## R1：手机端步骤式职业决策

目标是在窄屏职业模式中把一个超长内滚面板拆成明确步骤，同时保留桌面端现有布局。关键月使用四步，普通月自动跳过调查。

关键月流程：

> 调查 → 安排日程 → 构建观点 → 确认提交

普通月流程：

> 安排日程 → 构建观点 → 确认提交

### 任务清单

- [x] R1.1 定义窄屏步骤模型和当前步骤的 React 局部状态，不增加 `GameState` 字段。
- [x] R1.2 将组织压力、剩余时间和调查网络放入调查步骤。
- [x] R1.3 将三种日程放入独立步骤，并允许提交前重新选择。
- [x] R1.4 将三方研究线索、本话术语、研究承诺和方案选择放入构建观点步骤。
- [x] R1.5 增加 `draftDecision`。手机端点击方案只选择草案，不立即调用结算。
- [x] R1.6 增加确认页，汇总主题、剩余时间、日程、置信度、失效条件、自检完成数和方案摘要。
- [x] R1.7 最终确认后只调用一次 `makeDecisionWithSound`，随后沿用现有锁定和结算流程。
- [x] R1.8 增加粘性本月摘要和步骤指示，并为当前步骤使用 `aria-current="step"`。
- [x] R1.9 步骤切换后把焦点移动到新步骤标题，非活动步骤不得留在 Tab 顺序中。
- [x] R1.10 明确返回语义。调查消耗不回退，日程和研究承诺可以继续修改。
- [x] R1.11 处理视口旋转与跨越窄屏断点的情况，任何尺寸变化都不能自动提交草案。
- [x] R1.12 增加关键月完整旅程、普通月跳过调查、返回修改和单次提交的浏览器测试。

### 验收标准

- 390 × 844 下同时只显示一个步骤，没有横向溢出和嵌套长滚动区。
- 一月的全部调查节点、日程、承诺和方案都可通过键盘与触屏访问。
- 点击方案后 `state.locked` 仍为 `false`，历史记录数量不变。
- 最终确认后只产生一条历史记录，进入现有结算页。
- 返回调查页不会退还已经花费的时间块。
- 桌面职业模式继续使用当前纵向流程。
- 剧情模式、独立投委会、每日挑战和档案抽屉不进入步骤器。

### 暂不处理

- 不持久化未提交的步骤、草案和研究承诺。刷新后可以回到本话决策起点。
- 不增加滑动手势、页面路由或调查撤销。
- 不修改调查预算、调查奖励和评分公式。

- 不增加基金经理好感、合规声望等新数值条。
- 不新增组织玩法模式或组织角色立绘。

## R5：方法论视觉与办公室环境状态

R5 依赖 R3 的统一舞台契约。目标是通过构图、道具与环境反馈表达人物和历史，不扩充攻略角色。

### R5A：四名角色的方法论视觉

- [ ] R5.1 定义 `CharacterStageProfile`，包含人物位置、缩放、镜头距离、背景焦点、光色、道具层和粒子策略。
- [ ] R5.2 为林若宁增加纸质财报、便签、产业链图和暖光语法。
- [ ] R5.3 为陈星禾增加数据墙、订单流曲线、冷屏反光和偏侧站位。
- [ ] R5.4 为周明昭增加情景树、白板留白、稳定远镜头和低动态语法。
- [ ] R5.5 为赵承宇增加双屏、代码窗口、数据清洗表和协作空间。
- [ ] R5.6 工作场景减少统一漂浮星点，咖啡馆与关系场景才保留克制的柔和粒子。
- [ ] R5.7 在观点交锋中沿用同一图形语法，避免只依靠粉、蓝、紫、灰区分角色。

- [ ] R5.9 为所有透明道具资源增加角色编号映射、尺寸、压缩和授权记录。

验收要求：

- 隐去姓名和主题色后，仍能从构图与道具区分四人的研究方法。
- 陈星禾突出信号监控，赵承宇突出清洗、回测和协作，两者视觉表达不能同质化。
- 道具不会遮挡人物面部、对白面板和舞台状态标签。
- 手机、桌面和矮横屏都能看到至少一个方法论识别点。
- 不需要重画四人的全部姿势，也不通过道具暗示选项优劣。

### R5B：办公室环境状态

- [ ] R5.10 新增纯函数 `deriveOfficeVisualState(state, meta)`，不修改存档。
- [ ] R5.11 输出离散的整洁度、白板积累、透支、失败档案、关系空缺、周目残留和真相矛盾状态。

验收要求：

- 环境积累不会遮挡人物和对白，也不会恢复旧的计数叠层。
- 高疲劳可以通过杯子、屏幕、灯光或空椅表达，不只依赖数量增加。
- 关系遗憾使用克制的负空间或物件变化，不删除历史记录和真实计数。

- 减少动态效果时不运行非必要动画，关键信息不能只靠颜色传达。
