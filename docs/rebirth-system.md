# 重生、调查、研究室与因果回溯

本文档说明 2025 主线使用的跨周目系统。系统把未来记忆、关键月份调查、研究室痕迹、已读记录和互动影游式回溯连接起来。2023、2024 和 `demo` 继续沿用原流程。

年度剧情的新游戏会把 `experienceMode` 写入跨周目状态。职业模式展示本页说明的调查、研究室和因果回溯操作。剧情模式在主流程中隐藏职业系统并由会话层协助研究承诺，底层仍保留兼容关系分支和存档迁移所需的状态。

## 核心循环

关键月份会在研究选择前提供有限时间块。玩家可以调查公开材料、财务模型、产业访谈、量化信号、风险边界、档案记录或身体状态。

调查节点可以要求：

- 前置调查。
- 当前周目。
- 记忆钥匙。
- 研究捷径。
- 研究室整理结果。
- 剩余时间块。

完成调查会留下待验证线索、已验证证据、反例或状态记录。提交研究方案时，线索按照研究方法影响证据、清晰度、风险意识、反思价值和疲劳。

玩家可以提前提交。省下时间通常降低疲劳，证据缺口也会进入评分、人物反馈和跨周目档案。

## 关键月份

| 月份 | 主题 | 主要冲突 | 跨周目作用 |
|---|---|---|---|
| 一月 | 未来记忆审计 | 技术突破如何传导到收入、利润和竞争 | 建立单位经济性、失败样本和观察仓入口 |
| 四月 | 结果正确与证据完整 | 一季报证明结论后，当时证据是否已经充分 | 使用因果缺口审计事后正确偏差 |
| 七月 | 产业链与资金分化 | 基本面兑现和资金选择出现时间差 | 使用失败样本和捷径完成分层再平衡 |
| 九月 | 规则变化与组织权限 | 新规改变因子、成本和风险边界 | 把同事关系转化为闭门数据权限与共同校准协议 |
| 十二月 | 年度档案与真相路线 | 未来记忆来自亲历、标题或事后解释 | 审计全年档案并提出被污染的未来假设 |

其他月份继续承担资源积累、人物推进、方法训练和研究室物件增长。关键月份读取这些积累，避免每一话都扩展成相同规模的调查关卡。

## 记忆钥匙

完成年度结局并进入下一周目时，系统根据整年行为发放记忆钥匙：

| 钥匙 | 来源 | 下一周目作用 |
|---|---|---|
| 因果缺口 | 空降结论或低推理评分 | 解锁单位经济性、事后正确审计和证据链推演 |
| 样本污染 | 长期使用量化方法或追逐热度 | 解锁失败区间、失败样本和完整样本推演 |
| 身体记忆 | 高疲劳或透支路线 | 降低首次休息成本，并揭示疲劳对判断和关系的影响 |
| 机会成本 | 长期风险管理或平衡路线 | 解锁观察仓、分阶段验证和机会成本推演 |

钥匙不会标记标准答案。它改变玩家可以调查的问题，也会给旧时间线增加新的因果注释和推演方式。

## 研究捷径

人物关系和长期方法会在周目结束时转化为研究捷径：

- 林若宁的产业联系人簿降低产业访谈成本。
- 赵承宇的自动回测管线降低因子调查成本，并开放九月闭门失败样本。
- 周明昭的风险审查模板降低风险讨论成本。

七月的完整混合路线需要自动回测管线。普通时间预算无法同时完成业务热力图、失败样本和分层再平衡。捷径因此会改变可行路线。

## 研究室中枢

档案抽屉提供研究室标签。五类物件会读取本周目和跨周目状态：

| 物件 | 状态来源 | 整理结果 |
|---|---|---|
| 便签墙 | 基本面研究与协作次数 | 回看证据缺口、承诺和人物回调 |
| 白板 | 框架型研究次数 | 连接跨月假设，第二周目可能发现未重置字迹 |
| 咖啡杯 | 深度研究与加班次数 | 检查透支并恢复少量疲劳和生活平衡 |
| 研究档案柜 | 已完成月份 | 按假设和反例重排记录，开放十二月档案审计 |
| 研究捷径板 | 跨周目捷径 | 把联系人、回测管线和风险模板可视化 |

研究室整理只执行一次，并写入稳定旗标。后续月份可以使用这些旗标触发回调或调查前置条件。

## 因果回溯

档案抽屉的因果回溯标签提供三种模式。

### 观看模式

玩家先通过树状图查看主时间线和回溯分支。父子连接表示分支从哪条旧路线产生，节点会显示周目、完成月份、结局和分叉月份。选择一条路线后，可以查看十二个月状态、关键月锚点、调查路径、行动日志、实际研究判断和月度结果。观看不会修改游戏状态。

记忆钥匙会给旧锚点添加后来获得的理解。例如因果缺口会补出收入、利润和时点变量，样本污染会标记被筛掉的失败区间，身体记忆会显示疲劳影响，机会成本会显示没有行动的后果。

### 分叉模式

系统在一月、四月、七月、九月和十二月进入时保存月初快照。完成一条时间线的年度结局后，玩家可以从这些锚点创建新分支。

分叉遵守以下规则：

- 原时间线会在 12 条容量范围内保留。
- 恢复锚点当时的 `GameState` 和调查进度。
- 当前已经获得的记忆钥匙继续生效。
- 锚点之前形成的关系、失约、边界和职业状态原样保留。
- 内容版本变化后，旧锚点仍可观看，但不能恢复。
- 最多保存 12 条时间线。分叉达到上限时会被拒绝。重新开始会优先移除刚放弃的暂停路线，新周目会移除最旧的已完成路线，历次结局摘要继续保留。

分叉不会把人物变成密码锁。知道某个选择会伤害关系，不代表点击相反按钮就能自动证明长期可信。

### 推演模式

推演使用锚点状态和当前记忆钥匙计算一组反事实指标，用于比较补齐证据、恢复失败样本、建立观察仓或降低透支可能带来的差异。

推演结果不会修改当前 `GameState`、调查进度或时间线。它也不承诺市场结果和人物关系必然按预测发展。

## 时间线事件

活跃时间线会记录：

- 调查行动。
- 研究室整理。
- 日程选择。
- 研究判断。
- 回溯分叉。

事件日志用于解释分支如何形成。它不替代 `RoundResult`，月度结算仍由现有结算引擎负责。

## 已读跳过

对白在玩家实际推进后才写入已读记录。第二周目以后可以连续跳过已经读过的对白。跳过会停在首个新对白、未结算研究选择或月份边界。

跳过不会自动选择调查、日程或研究方案。

## 真相路线

第二周目取得因果缺口和样本污染后，十二月会出现未来记忆来源审计。玩家还需要：

1. 在研究室整理全年档案。
2. 比对未来记忆和后来标题。
3. 恢复失败月份与被筛选样本。
4. 把身体状态或机会成本写进框架。
5. 使用全部线索形成被污染的未来假设。

完成调查后会开放年度策略会专属方案。该方案写入 `rebirth_truth_route`，结局面板优先显示真相结局。

## 存储与迁移

单周目状态继续保存在：

```text
rebirthGameState:v2:<year>
```

跨周目状态保存在：

```text
rebirthMeta:v4:<year>
```

v4 增加 `experienceMode`，取值为 `romance` 或 `career`。体验模式在创建新游戏时确定，重新开始、跨入下一周目、恢复暂停分支和从关键月分叉都会沿用该值。

系统按顺序尝试读取 `rebirthMeta:v4`、`rebirthMeta:v3`、`rebirthMeta:v2` 和 `rebirthMeta:v1`。旧 v3 存档缺少体验字段时迁移为职业模式。旧 v2 存档会迁移为空时间线，并从当前进度开始记录。旧 v1 的一月调查会先迁移到按月份保存的调查结构。

快照只保存在关键月份月初。每条分支保存自己的分支头、调查状态、事件、锚点引用和结局。反事实推演单独保存，最多保留最近 32 条结果。

重新开始会暂停当前路线并创建一条新的同周目路线。完成年度结局会封存当前路线，再创建下一周目主线。刷新页面会恢复当前活跃分支和单周目存档。

浏览器清理站点数据后，本地时间线也会被删除。玩家可以在设置中导出完整存档、生成分享码，或使用自己的私密 GitHub Gist 保存加密存档。云同步会传输单周目状态、跨周目状态和时间线，具体安全边界见 [`platform-modes.md`](platform-modes.md)。

- `src/game/rebirthDecisionBonus.ts`：线索与研究方法的评分加成规则。
- `src/game/rebirthBranches.ts`：读取周目和记忆钥匙的剧情分支。
- `src/game/rebirthFlow.ts`：已读记录和连续跳过。
- `src/game/rebirthOffice.ts`：研究室物件、发现和状态变化。
- `src/components/InvestigationPanel.tsx`：首屏需要的关键月调查界面。
- `src/components/ArchiveDrawer.tsx`：按需加载的记录、档案和研究室抽屉。
- `src/components/RebirthPanel.tsx`：研究室和跨周目档案界面。
- `src/components/RebirthTimelinePanel.tsx`：按需加载的树状时间线、观看、分叉和推演界面。
- `src/rebirth.css` 与 `src/rebirth-v2.css`：首屏重生界面样式。`src/timeline.css` 随回溯面板异步加载。
- `src/app/useGameController.ts`：同步单周目、跨周目和时间线状态。
