# 稳定性、浏览器回归与无障碍

本文记录运行时恢复、浏览器端回归测试、键盘操作和社区内容包安全边界。功能设计与模式契约继续见 `platform-modes.md`。

## 运行时恢复

顶层应用由 `AppErrorBoundary` 保护。懒加载组件或渲染过程抛出未处理错误时，页面显示恢复界面，保留浏览器本地存档，并提供重新载入和返回年度剧情两个入口。

错误边界会派发 `rebirth-app-error` 浏览器事件，事件包含错误消息和 React 组件栈。项目当前不会自动上传该信息。

错误边界无法捕获事件处理函数、异步回调和浏览器外部脚本中的全部错误。新增异步流程时仍需要在业务边界处理失败状态。

## 键盘和焦点

页面提供「跳到主要内容」链接，键盘用户可以直接到达主菜单或当前玩法的主要区域。玩法页面还提供返回主菜单入口。

档案抽屉遵循模态对话框行为：

- 打开后焦点移到关闭按钮。
- `Tab` 和 `Shift+Tab` 保持在抽屉内部。
- `Escape` 关闭抽屉。
- 关闭后焦点回到打开档案的按钮。
- 标签页支持左右方向键切换。
- 当前标签页是唯一进入普通 Tab 顺序的标签按钮。

新增模态界面时应复用相同的焦点进入、限制、关闭和恢复规则。

## 单元测试与浏览器测试

Vitest 只运行 `src/` 下的单元和集成测试。`vitest.config.ts` 明确排除 `scripts/e2e/`，避免两个测试运行器收集同一文件。

浏览器测试位于：

```text
scripts/e2e/platform.spec.js
```

Playwright 配置位于：

```text
scripts/playwright.config.js
```

首次在本地运行：

```bash
npm run e2e:prepare
npm run test:e2e
```

Playwright 与 axe 使用 `package.json` 中锁定的开发依赖。`e2e:prepare` 下载对应 Chromium，浏览器二进制保存在 Playwright 的本机缓存中，不提交到仓库。

完整本地检查可以直接运行：

```bash
npm run check
```

该命令通过 `scripts/check.py` 顺序执行 Python、前端和 Chromium 回归。只排查浏览器问题时可以运行 `uv run python scripts/check.py --e2e`。

## 浏览器回归范围

当前 Chromium 回归覆盖：

- 主菜单的新游戏体验入口、继续游戏和兼容深链。
- 主菜单封面加载、宽屏分栏、窄屏排列和横向溢出。
- 年度剧情的跳过导航。
- 档案抽屉打开、关闭和焦点恢复。
- 剧情模式隐藏职业表单，职业模式保留完整研究流程。
- 三位同事的观点分开显示，窄屏可以滚动阅读完整内容。
- 研究室物件不覆盖主舞台，累积后会在研究室显示。
- 独立投委会完成五轮答辩并保存历史。
- 每日挑战保存首次结果，练习不覆盖首次记录。
- 内容工坊保存内容包后，案例进入投委会案例库。
- 平台模式加载失败时显示恢复界面。
- 主菜单与四种玩法的严重和致命 axe 规则。
- 年度剧情、投委会、每日挑战、内容工坊和设置面板的关键深色文字对比度。
- 观点卡和研究室概览在浅色、深色主题下的文字对比度。

Playwright 本地失败时会在 `test-results/` 保留截图、视频和跟踪文件。`CI` 环境才启用 HTML 报告。具体产物由 `scripts/playwright.config.js` 控制。

## 自动无障碍检查边界

axe 检查严重和致命问题。通用 axe 扫描关闭 `color-contrast` 规则，因为动态主题、背景图和半透明叠层会产生需要人工判断的结果。关键深色界面另有显式对比度测试，正文门槛为 4.5 比 1。

以下内容仍需要人工检查：

- 未纳入显式样本的浅色和深色主题文字对比度。
- 200% 字体缩放和浏览器缩放。
- Windows 高对比模式。
- `prefers-reduced-motion`。
- VoiceOver、NVDA 或其他真实读屏器。
- 小屏触控目标和软键盘遮挡。
- 中文语音提示的停顿和朗读顺序。

自动检查通过只说明没有命中当前规则集中的严重问题，不代表获得完整无障碍认证。

## 社区内容包安全边界

内容包继续使用版本 1 格式，并增加以下限制：

- 单个 JSON 内容包最多 256 KiB。
- 每个内容包最多 20 个案例。
- 每个案例最多 8 个研究方案，至少 2 个。
- 案例 ID 和方案 ID 在各自作用域内必须唯一。
- 标题、作者、说明、假设、业务事实和方案文本有长度上限。
- `updatedAt` 必须是可解析的日期时间。
- 评分字段必须是有限数字并位于原有范围内。

解析器在 `JSON.parse` 之前检查 UTF-8 字节数，避免超大输入先占用过多内存。保存到本地案例库前会再次检查序列化后的体积。

这些限制保护浏览器存储和投委会案例库，不构成内容事实审核。社区案例的来源、版权和金融事实仍由作者与导入者判断。

## 本地检查与发布

仓库不再使用 GitHub Actions 执行测试 CI。代码风格、类型检查、Vitest、Python、前端契约、品牌校验、包体预算和 Playwright 都由维护者在提交前运行 `npm run check` 完成。

本地检查还包含两层复杂度门禁。Ruff C901 与 ESLint `complexity` 的圈复杂度上限都是 15。`scripts/test_maintainability_metrics.py` 会核对 Python 长行、长函数、大文件和 C901 忽略项的当前基线，防止这些指标增加。

`.github/workflows/pages.yml` 只保留 GitHub Pages 必需的安装、Vite 打包、产物上传和发布步骤，在 `main` 更新或手动触发时运行。`npm run build:pages` 不执行 TypeScript 检查，这会继续消耗少量发布分钟，但不再为每个拉取请求重复运行测试矩阵。

Pages 工作流中的 `npm run build:pages` 只保证 Vite 发布产物可以生成，不能替代本地完整检查。测试失败记录和 Playwright 诊断保留在本机工作区，不上传到 GitHub Actions 产物。
