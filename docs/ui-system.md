# 界面布局与主题系统

本文记录主菜单、年度剧情、独立平台模式、设置弹层、存档和云同步的布局与配色规则。界面修改应优先遵守这些边界，避免单个页面的局部样式影响其他模式。

## 主菜单封面

主菜单顶部使用 `assets/key-art.webp`。封面遵守以下规则：

- 宽屏使用左右分栏，标题、说明和继续入口在左，人物画面在右。
- 840px 以下改为上下排列，封面在前，文字和操作在后。
- 图片使用固定宽高属性并优先加载，减少首屏布局跳动。
- 图片只营造氛围，设置空替代文字并从无障碍树中隐藏。
- 深色主题降低图片亮度和饱和度，文字仍放在实色面板上。
- 模式卡片不叠加人物图，入口名称和说明始终保持稳定对比度。

## 布局职责

### 年度剧情

年度剧情使用单视口布局：

- `.app-shell.mode-story` 占满 `100svh`。
- 主舞台、对白和操作按钮使用完整可用高度。
- 返回主菜单入口固定在顶部设置按钮旁，并为窄屏缩短文字。
- 返回入口不能覆盖 `.interaction-actions` 或设置按钮。
- 年度剧情内部可以滚动的区域由组件自己管理，例如对白、决策和设置弹层。

### 投委会、每日挑战和内容工坊

三种平台模式使用正常文档滚动：

- `body` 负责纵向滚动。
- `.platform-screen` 不设置固定视口高度。
- 返回主菜单入口位于页面顶部。
- 页面内容必须可以滚动到最后一个操作区域。
- 窄屏下不允许产生横向页面滚动。

不能再在 `html`、`body` 或 `#root` 上统一设置 `overflow: hidden`。剧情模式的裁切只允许通过 `.app-shell.mode-story` 实现。

## 返回主菜单

主菜单负责集中展示年度剧情、投委会、每日挑战和内容工坊。进入玩法后，页面只保留一个返回主菜单入口，减少常驻选项对当前任务的干扰。

- `BackToMenu` 在主菜单之外的四种玩法中统一显示。
- 年度剧情使用固定定位，并避开顶部设置按钮。
- 独立平台模式把入口放在正常页面流顶部。
- 窄屏年度剧情只显示返回箭头，仍保留完整无障碍名称。
- 返回入口需要有文字、边框和清晰焦点轮廓。

## 平台配色变量

公共变量位于 `src/platform-shell.css`，平台模式在 `src/platform-theme.css` 中再次声明最终值，确保旧基础样式不会覆盖深色主题：

- `--platform-bg`：页面背景
- `--platform-panel`：普通卡片
- `--platform-panel-strong`：输入框和高层卡片
- `--platform-panel-muted`：次级信息块
- `--platform-text`：主文字
- `--platform-muted`：说明文字
- `--platform-border`：边框
- `--platform-input`：表单背景
- `--platform-input-border`：表单边框
- `--platform-placeholder`：占位文字
- `--platform-disabled-bg`：禁用控件背景
- `--platform-disabled-text`：禁用控件文字
- `--platform-accent`：强调色
- `--platform-accent-soft`：强调色浅背景
- `--platform-focus`：键盘焦点轮廓

深色主题必须为这些变量提供独立值。不能只降低透明度来制造次级文字，因为透明文字叠在复杂背景上会迅速失去可读性。

## 模式视觉区分

三种平台模式共享卡片、输入框、按钮和排版语言，同时使用不同强调色：

- 投委会：粉紫色，突出人物追问和判断责任。
- 每日挑战：蓝紫色，突出日期、连续完成和固定约束。
- 内容工坊：薰衣草色，突出编辑、结构和预览。

强调色只用于导航、标签、选中状态、进度和焦点。正文仍使用统一的主文字与说明文字变量。

## 深色主题可读性

深色主题应满足：

- 普通正文与所在实体卡片的对比度至少为 4.5:1。
- 大字号标题至少为 3:1。
- 输入内容、标签和重要说明不能使用低透明度灰色。
- 占位文字必须可辨认，但不能与已输入内容混淆。
- 禁用按钮使用独立背景和文字颜色，不能只设置很低的整体透明度。
- 投委会暗色场景中的追问、回答说明和人物职位使用高亮浅色。

Playwright 会检查投委会正文、每日挑战正文、内容工坊字段标签和云同步安全说明的关键对比度。

## 观点卡与研究室

三方观点使用 `DebatePanel` 分别渲染。每位角色拥有独立的标题、方法标签和正文卡片，剧情模式与职业模式使用同一结构。档案抽屉回看观点交锋时也按角色保留三条内容。

办公室物件概览位于档案抽屉的研究室标签。主舞台不根据 `OfficeState` 叠加便签、白板或咖啡杯，避免状态装饰干扰人物画面和对白。

## 设置、存档和云同步

设置弹层采用渐进展开：

1. 年份、主题、音频和指标模式直接显示。
2. 存档与跨设备转移默认折叠。
3. 展开后分为本地文件、分享码和加密云同步。
4. 云同步再次折叠，只有主动配置时才显示 GitHub 访问令牌、口令和 Gist 编号。

这样可以减少剧情过程中设置面板的高度，也能把普通本地存档和高级云同步区分开。

### 本地文件

用于导出存档、导入文件和导出游玩记录，不需要账号。

### 分享码

用于临时设备之间手动复制。文本域使用等宽字体，并保留清晰标签。

### 加密云同步

云同步使用安全卡片样式：

- 标题说明私密 GitHub Gist。
- `AES-GCM` 作为状态徽标显示。
- GitHub 访问令牌、同步口令和 Gist 编号使用统一深色输入框。
- 安全说明使用强调色左边框和实体背景。
- GitHub 访问令牌与同步口令不写入本地存储。

## 响应式规则

### 780px 以下

- 设置弹层变为视口内的固定面板。
- 投委会、每日挑战和内容工坊改为单列。
- 内容工坊侧栏取消粘性定位。
- 云同步表单改为单列。

### 480px 以下

- 内容工坊评分和选择器改为单列。
- 卡片内边距缩小。
- 页面不允许横向裁切。

## 滚动条

平台页面、案例列表、内容工坊本地案例库和设置弹层使用细滚动条。实时预览跟随页面文档滚动，不再制造额外内嵌滚动区。滚动条颜色来自 `--platform-border`，不能隐藏滚动条，因为用户需要知道仍有未显示内容。

## 浏览器回归

`scripts/e2e/platform.spec.js` 检查：

- 主菜单封面可以加载，宽屏保持左右分栏，窄屏保持图片在上且没有横向溢出。
- 返回主菜单入口不遮挡剧情操作按钮。
- 投委会、每日挑战和内容工坊在 640px 高视口可以滚动到底部。
- 390px 宽视口没有横向裁切。
- 深色主题关键文字达到最低对比度。
- 三位同事的观点卡分开显示，窄屏可滚动阅读。
- 研究室物件不覆盖主舞台，累积后会呈现便签、白板和咖啡杯。
- 设置弹层中的存档和云同步可以展开并阅读。
- 原有投委会、每日挑战、内容工坊、错误恢复和无障碍旅程继续通过。

## 主要文件

- `src/start-menu.css`：主菜单封面、模式卡片和返回入口
- `src/platform-shell.css`：公共变量、设置、存档和云同步
- `src/settings-polish.css`：设置基础控件布局
- `src/platform.css`：平台模式基础结构
- `src/platform-polish.css`：平台滚动、深色主题和视觉覆盖
- `src/platform-theme.css`：懒加载后的最终主题变量和滚动规则
- `src/App.tsx`：模式类名和平台样式加载顺序
- `src/components/SaveTransferPanel.tsx`：渐进式存档界面
- `src/components/CloudSyncPanel.tsx`：渐进式云同步界面
- `src/components/DebatePanel.tsx`：按角色分开的观点卡片
- `src/components/ArchiveDrawer.tsx` 与 `src/components/RebirthPanel.tsx`：研究室入口和物件概览
- `scripts/e2e/platform.spec.js`：布局、滚动、窄屏和对比度回归
