# 操作与界面

主游戏使用单视口布局。剧情舞台、对白、日程选择、研究方案和推进按钮都位于浏览器可视区域内，页面本身不滚动。内容较多时，仅对白或决策面板内部滚动。

## 主流程

每一话按以下顺序进行：

1. 阅读对白和未来记忆。
2. 查看三位同事的观点交锋。三种方法分别显示，不由当前角色代读。
3. 安排本话日程。
4. 选择一个研究方向并立即结算。
5. 查看同事复盘，再进入下一话。

底部操作栏是唯一的剧情推进区域。界面不会同时显示两组功能相同的继续按钮。

## 回看规则

`上一句` 允许玩家在结算前返回当前话的上一段对白。研究方案结算后，按钮会禁用，避免回退时静默撤销关系、评分和旗标。

已结算内容可以在 `记录与档案` 抽屉中查看。抽屉分为本话记录和研究档案两个标签，不改变当前游戏状态。完成十二话后，主操作面板会直接显示年度结局和研究图鉴。玩家确认复盘结果后，可以开启新周目。

键盘操作：

- `←`：返回上一段对白。
- `Enter` 或空格：推进当前剧情。

输入控件、按钮、设置菜单或档案抽屉获得焦点时，不触发这些快捷键。

## 自动保存

每个年份线分别保存在浏览器本地。刷新页面或切换回来时，会恢复该年份最近一次有效状态。重新开始会覆盖当前年份的存档，其他年份不受影响。

本地存档属于便利功能，不承担跨设备同步。浏览器隐私模式、存储配额不足或清理站点数据都可能让存档失效。

## 观点交锋

2025 年线的竞争假设按角色方法论拆分：

- 林若宁使用产业链、订单、收入质量和留存等基本面证据。
- 陈星禾使用量价、资金持续性、因子暴露和拥挤度等信号。
- 周明昭使用估值、压力测试、下行情景和退出条件等风险框架。

三张观点卡分别给出有证据支撑的假设。收束文案提醒玩家检查证据和盲区，不暗示唯一答案。

## 赵承宇立绘

赵承宇与三位女同事使用同一套细腻二次元画风、相近的人物比例和统一的舞台光影。三张透明 WebP 立绘分别表现中性、思考和释然状态，统一由 PixiJS 舞台加载，不再执行运行时白底抠图。
