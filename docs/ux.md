# 操作与界面

无参数访问先显示主菜单。进入年度剧情后使用单视口布局，剧情舞台、对白、剧情或研究选项和推进按钮都位于浏览器可视区域内。内容较多时，仅对白、决策面板或档案抽屉内部滚动。

## 主菜单

主菜单把入口按意图分成三组：

- 开始新游戏：剧情模式与职业模式。
- 挑战中心：独立投委会与每日挑战。
- 创作与扩展：内容工坊和本地优先说明。

继续游戏读取浏览器中的 2025 存档及其体验模式。新游戏卡片会创建对应体验的全新进度。进入任何玩法后，页面提供返回主菜单入口，不需要在年度剧情底部长期显示所有平台功能。

主菜单的主题封面只出现在顶部主视觉区。宽屏采用左侧标题与继续入口、右侧封面图的分栏，窄屏把图片移到文案上方。图片只负责营造氛围，不承载按钮或必要信息。模式卡片继续使用稳定的实色面板，避免人物画面干扰入口识别和深色主题可读性。

年度剧情直接链接为 `?mode=story`。新剧情模式使用 `?mode=story&play=romance&new=1`，新职业模式使用 `?mode=story&play=career&new=1`。旧的 `year`、`pixivn`、`pixi` 和 `staticStage` 深链继续直接打开年度剧情。

## 职业模式主流程

每一话按以下顺序进行：

1. 阅读对白和未来记忆。
2. 查看三位同事的观点交锋。
3. 关键月份使用有限时间块调查证据、反例或状态。
4. 安排本话日程。
5. 比较研究方案的主张、押注变量和主要代价。
6. 查看当前日程、推荐日程与最大风险，确认后提交本月判断。
7. 先查看行动因果，再按需展开完整评分复盘。

职业模式顶部每月只突出一个研究目标、一个主要风险和一个校准问题。研究可信度、团队信任、生活平衡与人物关系继续保留，但不要求玩家同时把全部底层指标当作当月任务。

一月、四月、七月、九月和十二月会显示调查网络。调查卡会标出时间成本、前置条件、记忆钥匙、研究捷径和证据可靠度。界面优先显示当前可以执行和已经完成的节点，需要前置条件或记忆钥匙的节点收进待解锁区域，玩家仍可展开查看完整网络。玩家可以保留未使用时间，直接进入日程和研究选择。

第一周目一月先展示公开材料、企业访谈和风险反例三条调查路径。其他当前可执行节点进入进阶调查区域，规则和奖励保持不变。第一话的日程先并列展示深度研报与生活优先，团队协作保留在可展开区域，帮助新玩家先理解投入与状态的取舍。

研究承诺默认先显示研究助理和折叠的手动配置。第一周目一月由研究助理自动采用 70% 基准判断、保留当前失效信号并完成三项自检。后续月份仍可用一次操作采用同一稳健模板。需要自行校准置信度、失效条件或审查项时，可以展开手动调整。最终评分、旗标和月度复盘继续读取同一份研究承诺，没有额外的隐藏奖励。

职业方案卡使用稳定的方法视觉和信息顺序，依次说明主张、押注变量、主要代价与推荐日程。点击方案先打开确认区，不会立即结算。推荐日程可以一键应用，玩家仍可返回上方手动调整。确认区只汇总已有选择，不增加新的研究问题。

方案或观点中出现 ARR、Barra 归因、Alpha 衰减、因子拥挤度、订单流、估值分位、DCF、单位经济性和敏感性分析时，界面提供当前内容实际命中的简短解释。术语说明只解释概念，不提示高分方案、关系收益或正确答案。

职业结算先显示做得好的地方、留下的风险和产生的后果。完整评分、等级评语和研究复盘默认折叠。展开后可以查看证据、清晰、风险、沟通、生活、组合、方法质量、业务结果、推理摘要、总分和等级。业务事实结算与知识卡继续保留。试玩记录增加方案预览时长、确认提交、推荐日程采用、术语展开、调查扩展、评分展开和第一话完成事件，数据仍只保存在玩家浏览器本地。

底部操作栏是唯一的剧情推进区域。界面不会同时显示两组功能相同的继续按钮。

对白正文按空行拆成独立段落。动作描写和角色说出口的话分别显示，不使用括号或引号重复标记。档案抽屉沿用相同的分段方式。

## 剧情模式主流程

剧情模式把操作集中在对白和人物选择：

1. 阅读对白与人物交锋。
2. 从章节、心动焦点和相处节奏了解当前状态。
3. 选择如何回应眼前的人。
4. 阅读对白式结果并继续下一话。

调查网络、研究线索、日程选择、研究承诺和精确职业指标不会出现在剧情模式的主决策界面。辅助提示会说明研究细节由系统采用稳健方案。人物对诚实、承诺与边界的反应仍然保留，剧情模式不会把所有选项变成相同结果。

## 回看与已读跳过

`上一句` 允许玩家在结算前返回当前话的上一段对白。研究方案结算后，按钮会禁用，避免回退时静默撤销关系、评分和旗标。

对白在玩家实际点击继续时写入跨周目已读记录。第二周目以后，当前节点已经读过时会启用 `跳过已读`。连续跳过会停在以下位置：

- 首个没有读过的对白。
- 研究选择节点。
- 当前月份的末尾。

跳过不会自动选择日程、调查或研究方案，也不会跨过尚未结算的月份。

键盘操作：

- `←`：返回上一段对白。
- `Enter` 或空格：推进当前剧情。

输入控件、按钮、设置菜单或档案抽屉获得焦点时，不触发这些快捷键。

## 档案与研究室

职业模式的 `档案与研究室` 抽屉包含四个标签：

| 标签 | 内容 | 是否改变状态 |
|---|---|---|
| 本话记录 | 当前月份已经阅读的对白与研究节点 | 否 |
| 研究档案 | 关系、月度研究札记、知识卡和重生档案 | 否 |
| 因果回溯 | 时间线、关键月锚点、行动日志、因果注释和反事实推演 | 分叉和恢复会改变状态 |
| 研究室 | 便签、白板、咖啡杯、档案柜和研究捷径板 | 首次整理物件时会改变状态 |

研究室物件会根据本周目积累开放。已经整理的物件会保留发现内容，不允许重复领取状态变化。

便签、白板和咖啡杯会出现在研究室的房间概览中。档案柜和研究捷径板作为可整理的物件卡片显示在下方。主舞台只呈现人物与场景，不叠加物件计数装饰，减少对白区域附近的视觉干扰。

档案抽屉不进入首屏 JavaScript。鼠标移入或键盘聚焦 `档案与研究室` 时会预加载抽屉，真正打开后才下载档案代码。因果回溯面板和时间线样式继续单独按需加载。

完成十二话后，主操作面板会显示年度结局和研究图鉴。玩家确认复盘结果后，可以带着记忆钥匙、研究捷径、异常、已读记录和旧时间线进入下一周目。

剧情模式的入口名为 `剧情回顾`，只提供 `本话记录` 和 `心动档案` 两个标签。该模式不显示因果回溯和研究室，玩家可以直接回看对白和人物关系。

## 因果回溯操作

因果回溯标签先显示父子分支树。主时间线是树干，从关键月建立的路线作为子节点展开。选择一条分支后，界面默认只显示关键月和当前月，也可以切换为全部十二个月。

### 观看

选择带有回溯锚点的关键月份后，可以查看月初状态、关系摘要、实际选择和后来获得的因果注释。观看不会改变存档。

### 分叉

完成一条路线的年度结局后，锚点详情会开放创建分支。分叉恢复该月月初状态，原路线继续保留。创建分支后，档案抽屉可以关闭，主界面从该月份继续游玩。

只有已完成路线的锚点可以分叉。剧情内容版本变化后，旧锚点保留观看能力，恢复按钮会禁用。

### 恢复暂停路线

重新开始或切换到其他分支时，当前路线会保存为暂停状态。时间线卡片上的 `继续这条线` 会恢复该分支头、调查进度和剧情位置。

### 推演

锚点详情会根据已获得的记忆钥匙显示推演选项。推演结果只显示指标对照和解释，不会修改当前月份、关系、调查或结局。

## 自动保存

每个年份线的单周目状态分别保存在浏览器本地。跨周目状态使用独立存档，保存体验模式、周目编号、记忆钥匙、研究捷径、调查、已读节点、研究室发现和时间线。

刷新页面或切换回来时，会恢复该年份最近一次有效状态。重新开始会覆盖当前单周目进度，同时把旧路线保存为暂停分支。其他年份不受影响。

本地存档是默认方式。浏览器隐私模式、存储配额不足或清理站点数据都可能让存档失效。玩家可以导出 JSON、复制分享码，或在设置中使用自己的私密 GitHub Gist 进行加密云同步。项目没有自建账号系统，访问令牌和同步口令不会写入本地存储。

体验模式随存档保存。地址中的 `play` 只为没有模式信息的新存档提供默认值，`new=1` 才会明确重建进度。页面消费该参数后立即移除它，避免刷新重复开局。

## 主题与可读性

主菜单、年度剧情、投委会、每日挑战和内容工坊共用语义化的前景色、面板色、输入框色和强调色。深色主题下，利益相关方压力卡、研究承诺、置信度选项、失效条件、自检项、分享码和云同步警告都有独立可读性回归样本。

主题保存在浏览器本地。浏览器测试对关键正文使用至少 4.5 比 1 的对比度门槛，半透明背景与动态舞台仍需要人工核对。

## 观点交锋

2025 年线的竞争假设按角色方法论拆分：

- 林若宁使用产业链、订单、收入质量和留存等基本面证据。
- 陈星禾使用量价、资金持续性、因子暴露和拥挤度等信号。
- 周明昭使用估值、压力测试、下行情景和退出条件等风险框架。

三张观点卡分别给出有证据支撑的假设。每张卡只显示一位角色的姓名、方法标签和观点，避免把多人台词拼成一个对白段落。收束文案提醒玩家检查证据和盲区，不暗示唯一答案。观点中命中的本话术语会在卡片下方以默认折叠的列表解释。

## 赵承宇立绘

赵承宇与三位女同事使用同一套细腻二次元画风、相近的人物比例和统一的舞台光影。三张透明 WebP 立绘分别表现中性、思考和释然状态，统一由 PixiJS 舞台加载，不再执行运行时白底抠图。
